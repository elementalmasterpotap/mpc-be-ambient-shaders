# Workflow — деплой и патчноты

## Деплой
После любого изменения шейдера: `.\install.ps1 -NoPrompt`
Копирует в `%APPDATA%\MPC-BE\Shaders\`, генерирует Ready-файлы.

## Патчнот → начало `docs/PATCHNOTES.md`
Писать: новые эффекты, изменение грейдинга/vibrance/gain, фиксы артефактов.
НЕ писать: рефакторинг без визуала, комментарии, структурная уборка.
`docs/TELEGRAM_POST.md` — оригинальный анонс, не редактировать.

Структура:
```
## Патч vX.X.X — YYYY-MM-DD
*Tagline — одна строка суть*

[Симптом / контекст — что было не так или зачем патч]

[Что сделал — что пользователь теперь увидит]

[Список с тире — только если несколько отдельных изменений]
```

Правила тона:
- От первого лица: "убрал", "прикрутил", "поймал"
- Без технических терминов: warm tint→тёплый оттенок · halation→свечение у края · grading→цветокоррекция · vibrance→насыщенность
- Tagline обязателен (курсив)
- Финальная фраза-вывод желательна
- Неформально, мат ок
