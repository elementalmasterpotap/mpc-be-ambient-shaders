# Development Rules & Decision Log

Оперативные правила — в `CLAUDE.md`. Здесь фиксируются принятые решения и их причины.

## Правила (кратко)

- Детектор (`IsDarkColumn`, `IsDarkRow`, пороги, веса) — **не трогать**.
- После каждого изменения шейдера: `.\install.ps1 -NoPrompt`.
- Приоритет: максимальное визуальное качество + насыщенные цвета.
- Работаем только с SM3 (`ProfessionalLighting_SM3.hlsl`).

## Лог решений

### Добавление новых функций-хелперов (сломано, откат)
Попытка добавить блок новых функций (`Vignette`, `ChromaticAberration`, `VerticalEdgeGradient`, `LensDirt` и др.) перед `BuildSideFillSM3` сломала детектор.
Причина не установлена точно — вероятно превышение лимита инструкций ps_3_0 или конфликт регистров при большом числе сэмплов.
**Вывод:** новые эффекты добавлять по одному, с проверкой детектора после каждого. Не добавлять несколько тяжёлых функций одним коммитом.

### Детектор — 5 точек вместо 3 (сломано, откат)
Попытка расширить IsDarkColumn/IsDarkRow с 3 до 5 точек (0.15/0.35/0.50/0.65/0.85) и добавить зонды xBorder 0.111/0.117 сломала детектор.
Причина: вероятно превышение лимита инструкций ps_3_0 — каждый вызов IsDarkColumn теперь делает 5 tex2D вместо 3, а зондов стало больше.
**Вывод:** топология детектора (количество точек и зондов) не менять. Детектор — замороженная зона.

### Детектор — веса точек (сломано, откат)
Попытка снизить вес центральной точки (`b * 0.20` вместо `/ 3.0`) для защиты от логотипов.
Результат: детектор перестал срабатывать. Откат к оригиналу `(a + b + c) / 3.0`.
**Вывод:** веса и количество точек детектора не менять.

### centerLum guard — мерцание при тёмных сценах
Первая версия: `centerLum > 0.04` от одной точки `(0.5, 0.5)` — мерцание на тёмных сценах.
Исправление: `max` из 5 точек центра кадра, порог `> 0.02`.

### edgeFloor SM3
Снижен с `0.34/0.16` до `0.18/0.10` — убирает серый остаток у краёв экрана.

### SoftClip удалён из GradeAmbientSM3
Дублировал Filmic. Убран — остался только Filmic в конце.

### ArtifactGuardSM3 — убран if
`if (lum > lumCap)` заменён на `min(1.0, lumCap / max(lum, 1e-4))` — без ветвления.

### spreadAmt — исправлена тавтология
`0.18*(1-darkScene) + 0.18*darkScene` всегда равно 0.18. Упрощено до `0.22 + 0.16*n + 0.24*streakRisk`.

### SaturationLimit — Chroma вместо length
`length(d)` даёт евклидову норму, не соответствующую перцептивному насыщению.
Заменено на `Chroma(col)` (max-min). Равномернее по оттенкам.

### WarmHalation — порог с 0.38 до 0.18
Галация теперь работает и при тёмных сценах с ярким объектом у края.

### Новые эффекты — добавлены по одному, детектор цел

Порядок добавления: Vignette → ChromaticAberration → VerticalGradient → LensDirt.
Все реализованы без новых texture samples (чистый ALU) — это ключевое условие безопасности для ps_3_0.
- **Vignette**: `pow(dot(tc,tc), 3.0) * 0.55`, сила lerp 0.40
- **ChromaticAberration**: R/B сдвиг через luma-градиент, сила 0.18, квадратичный рост к углам
- **VerticalGradient**: верх темнее низа, сила 0.38 (~21% max затемнение)
- **LensDirt**: два уровня Hash12 (0.018 + 0.055 scale), маска по яркости, сила 0.016
**Вывод:** эффекты без tex2D безопасны для детектора даже при добавлении группой.

### GradeAmbientSM3 — lumVib после десатурации
`lum` пересчитывается после десатурации перед применением vibrance (`lumVib`).
Vibrance теперь корректно пивотирует на актуальной яркости.
