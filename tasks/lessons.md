# Lessons Learned — MPC-BE Ambient Shaders

Уроки из реальных поломок. Читать перед любым изменением детектора или добавлением новых функций.

---

## КРИТИЧЕСКИЕ (сломали детектор, откат)

### L1 — Несколько tex2D функций за один раз → детектор ломается
**Что случилось:** добавили Vignette + ChromaticAberration + VerticalGradient + LensDirt одним блоком.
**Симптом:** детектор перестал срабатывать.
**Причина:** ps_3_0 instruction limit / register pressure при добавлении нескольких текстурных функций.
**Правило:** новый эффект с `tex2D` — добавляем по одному, проверяем детектор после каждого.

### L2 — 5 точек детектора вместо 3 → детектор ломается
**Что случилось:** расширили пробные точки: `{0.15, 0.35, 0.50, 0.65, 0.85}` вместо `{0.20, 0.50, 0.75}`.
**Симптом:** детектор перестал срабатывать полностью.
**Причина:** превышение бюджета texture sample slots в ps_3_0.
**Правило:** топология детектора (число точек + координаты) — ЗАМОРОЖЕНА навсегда.

### L3 — Изменение весов детектора → детектор ломается
**Что случилось:** снизили вес центральной точки с `/ 3.0` на `* 0.20` (для защиты от лого).
**Симптом:** детектор перестал срабатывать полностью.
**Причина:** веса критичны для корректного усреднения; изменение нарушило баланс.
**Правило:** веса в `IsDarkColumn` / `IsDarkRow` — ЗАМОРОЖЕНЫ. Только `xBorder`/`yBorder` в блоке уточнения границы.

## ИСПРАВЛЕНИЯ (баги, решены)

### L4 — centerLum: мерцание на тёмных сценах
**Проблема:** одна точка `(0.5, 0.5)` с порогом `> 0.04` → мерцание при тёмных сценах с одним ярким пикселем.
**Решение:** max из 5 точек центра кадра, порог снижен до `> 0.02`.
**Итог:** стабильный guard против чёрных кадров.

### L5 — edgeFloor: серый остаток у краёв экрана
**Проблема:** `FADE_FLOOR_LIGHT = 0.34`, `FADE_FLOOR_DARK = 0.16` давали видимый серый ореол.
**Решение:** снижено до `0.18` / `0.10`.
**Итог:** края чистые, fade уходит в чёрный.

### L6 — spreadAmt: тавтология
**Проблема:** `0.18*(1-darkScene) + 0.18*darkScene` всегда равно `0.18` (не зависит от сцены).
**Решение:** `0.22 + 0.16*n + 0.24*streakRisk` — реальная зависимость от расстояния и риска полос.

### L7 — SaturationLimit: евклидова vs перцептивная
**Проблема:** `length(d)` давал неравномерное распределение по оттенкам.
**Решение:** заменено на `Chroma(col) = max(R,G,B) - min(R,G,B)` (перцептивная насыщенность).
**Итог:** более равномерный предел по всему цветовому кругу.

### L8 — WarmHalation: порог активации
**Проблема:** порог `smoothstep(0.38, 0.85, luma)` не давал свечению работать в тёмных сценах.
**Решение:** снижен до `smoothstep(0.18, 0.85, luma)`.
**Итог:** halation активируется даже на тёмных сценах с ярким краем.

### L9 — lumVib: пересчёт lum после десатурации
**Проблема:** vibrance пивотился на исходный lum ДО десатурации → неправильное поведение.
**Решение:** `lumVib` пересчитывается из цвета ПОСЛЕ десатурации, перед применением vibrance.
**Итог:** vibrance корректно усиливает ненасыщенные каналы.

### L10 — SoftClip: дублирование Filmic
**Проблема:** SoftClip дублировал функциональность ACES Filmic tone mapping.
**Решение:** SoftClip удалён, остался единственный компрессор — Filmic.

### L11 — ArtifactGuardSM3: branching hazard
**Проблема:** `if (lum > lumCap)` создавал branch penalty в GPU шейдере.
**Решение:** `min(1.0, lumCap / max(lum, 1e-4))` — branchless, работает быстрее.

### L12 — Синие тона в ярких сценах (sky loss)
**Проблема:** яркие сцены агрессивно применяли warm tint → терялся синий канал; neutral blur overlay 18% убивал оставшийся синий.
**Решение:** blue в warm tint поднят с `0.95` → `0.98`; neutral blur overlay снижен с `0.18+0.24` → `0.10+0.18`.

### L13 — Паразитный синий мультипликатор в тёмных сценах
**Проблема:** 11% лишний blue multiplier в тёмных сценах давал голубоватый сдвиг.
**Решение:** убран из GradeAmbientSM3.

## БЕЗОПАСНЫЕ ПАТТЕРНЫ

### P1 — ALU-only эффекты безопасны в любом количестве
Vignette, ChromaticAberration, VerticalGradient, LensDirt — все реализованы без `tex2D`.
Можно добавлять группой, детектор не затрагивается.

**Формулы (справочно):**
- Vignette: `pow(dot(tc, tc), 3.0) * 0.55` (aspect-corrected)
- ChromaticAberration: квадратичный R/B split к углам по luma
- VerticalGradient: линейный градиент по Y (0=верх темнее, 1=низ)
- LensDirt: two-octave Hash12 noise, маскируется по яркости

### P2 — Откат при поломке детектора
```
cp docs/ProfessionalLighting_SM3.backup.hlsl shaders/ProfessionalLighting_SM3.hlsl
.\install.ps1 -NoPrompt
```

## SESSION LESSONS (добавлять после каждой правки)

### 2026-02-28 — "коричневое стекло": что вызывает и как лечить
**Симптом:** свечение выглядит тёплым но грязным — как через коричневое/рыжее стекло.
**Причина:** комбинация трёх факторов:
1. Слишком высокий `warm` вектор (`1.15/1.08/0.98`) — R+15%, G+8% давали красно-жёлтый налёт
2. Низкий порог активации warm (`edgeLum > 0.26`) — полутёмные сцены тоже получали warm tint
3. Высокий mix weight (`0.55*split + 0.45*w`) — warm применялся почти на 100%
**Решение (итеративно):**
- `warm`: `1.15/1.08/0.98` → `1.07/1.03/0.99` → `1.04/1.01/1.00`
- Порог `w`: `edgeLum - 0.26` → `edgeLum - 0.38`
- Mix weight: `0.55/0.45` → `0.46/0.36`
- `lerp(srcCol, col, ...)`: `0.97/0.90` → `0.90/0.82`
- `HALATION_AMOUNT_BASE`: `0.045` → `0.025`, FAR: `0.035` → `0.018`
**Правило:** если warm tint "грязнит" — сначала снижать `warm` вектор и порог `w`, потом mix weight, потом halation.

### 2026-02-28 — шейдер на пределе ps_3_0 instruction limit
**Симптом:** подсветка исчезает полностью после добавления ALU инструкций.
**Причина:** шейдер уже на пределе бюджета ps_3_0. Даже 4-6 новых ALU в функции которая вызывается дважды (BuildSideFill + BuildTopFill) → instruction limit превышен → шейдер не компилируется.
**Что прошло:** минимальные добавления (1-3 инструкции в одном месте). Что не прошло: blueDom+lerp в WarmHalation (+8), smoothstep+fadeN (+4×2), colNorm+lerp в bloom (+6×4 вызовов GradeAmbientSM3).
**Правило:** больше ничего не добавлять без удаления равноценного. Любое новое улучшение = сначала найти что можно убрать или упростить.

### 2026-02-28 — изменение сигнатуры функции → шейдер не компилируется → подсветка исчезает полностью
**Симптом:** подсветка исчезает полностью (не "не срабатывает детектор", а именно пусто).
**Причина:** изменили сигнатуру функции (добавили параметр texY в GradeAmbientSM3, edgeCol в WarmHalation) — если хотя бы один вызов не обновлён или компилятор не принял изменение, шейдер не компилируется вообще.
**Правило:** не менять сигнатуры функций. Вместо этого — inline ALU в BuildFill, или использовать уже доступные локальные переменные внутри функции (glowCol → blueDom внутри WarmHalation, tex.y применять после вызова GradeAmbientSM3 снаружи).
**Откат:** `cp docs/ProfessionalLighting_SM3.backup.hlsl shaders/... && install.ps1 -NoPrompt`

### 2026-03-01 — install.ps1 не компилирует шейдер — MPC-BE компилирует в runtime
**Симптом:** install.ps1 завершается успешно, но подсветка пропадает после запуска MPC-BE.
**Причина:** install.ps1 только копирует файл. Реальная компиляция ps_3_0 происходит в MPC-BE при загрузке шейдера через DirectX. Если шейдер не компилируется — подсветка просто исчезает без ошибки.
**Правило:** успешный деплой ≠ рабочий шейдер. Нужно проверять MPC-BE после каждого деплоя.

### 2026-03-01 — colNorm+lerp в bloom не влезает, fadeN влезает
**colNorm+lerp в bloom:** +6 ALU в GradeAmbientSM3 × 4 вызовов = +24 суммарно → не компилируется.
**smoothstep+fadeN:** +4 ALU в каждой BuildFill = +8 суммарно → влез, работает.
**Что делает fadeN:** `smoothstep(0.0, 0.06, n)` — плавный старт fade у шва, убирает жёсткий pop на n=0.
**Правило:** colNorm в GradeAmbientSM3 заблокирован навсегда — слишком дорого (×4 вызовов).

### 2026-03-01 — blueDom в WarmHalation не влезает в instruction limit
**Симптом:** подсветка исчезла после добавления blueDom (+8 ALU в WarmHalation).
**Причина:** шейдер на пределе instruction budget. +8 ALU в WarmHalation (вызывается 1× в каждой BuildFill = +16 суммарно) → не компилируется.
**Что влезло:** `SoftClip` удалён + `SaturationLimit` branchless + `VibranceBoost` без `sign` — всё применено, работает.
**Правило:** blueDom в WarmHalation — заблокировано навсегда пока не найдётся место.

### 2026-03-01 — оптимизации без потери качества — применены
**Что сделано:**
- `SoftClip` удалён — мёртвый код, нигде не вызывался (L10 знало, но функция торчала)
- `SaturationLimit` branchless: `(sat > maxSat) ? k : 1.0` → `min(1.0, maxSat/max(sat, 1e-4))` — аналог L11
- `VibranceBoost` убран `sign(strength)` — strength всегда > 0, sign был бесполезен
**Итог:** визуально идентично, чуть меньше ALU pressure. Работает.
